2. Um índice em um banco de dados relacional garante a integridade referencial entre tabelas.

a) Certo

b) Errado

✅ Gabarito: b) Errado

Índices aceleram consultas, mas não garantem integridade. A integridade referencial é garantida por chaves estrangeiras (foreign keys).

CESPE – MPU – 2022

3. Em uma transação de banco de dados, a propriedade de isolamento garante que, mesmo em caso de falha, o banco retorne a um estado consistente.

a) Certo

b) Errado

✅ Gabarito: b) Errado

Isolamento refere-se à execução concorrente de transações. A propriedade que garante retorno a um estado consistente após falha é a durabilidade (D de ACID).

4. O comando DELETE em SQL remove permanentemente uma tabela do banco de dados.

a) Certo

b) Errado

✅ Gabarito: b) Errado

O comando DELETE remove linhas de uma tabela. Para remover a tabela inteira, usa-se DROP TABLE.

CESPE – INSS – 2022

5. O modelo de dados NoSQL é sempre mais adequado que o relacional para aplicações governamentais com dados estruturados.

a) Certo

b) Errado

✅ Gabarito: b) Errado

Bancos relacionais são mais adequados para dados estruturados com integridade e transações ACID. NoSQL é ideal para dados não estruturados ou alta escalabilidade.

6. A normalização elimina completamente a redundância de dados em um banco de dados.

a) Certo

b) Errado

✅ Gabarito: b) Errado

A normalização reduz a redundância, mas não a elimina completamente. Algumas redundâncias podem ser mantidas para desempenho (desnormalização).

CESPE – TRE-SP – 2022

7. Uma view em SQL é um objeto que armazena dados fisicamente no banco de dados.

a) Certo

b) Errado

✅ Gabarito: b) Errado

Uma view é uma consulta virtual que não armazena dados fisicamente. Ela exibe dados de uma ou mais tabelas com base em uma consulta SQL.

CESPE – MPU – 2023

8. O comando TRUNCATE em SQL remove todas as linhas de uma tabela, mas mantém sua estrutura.

a) Certo

b) Errado

✅ Gabarito: a) Certo

TRUNCATE é um comando DDL que remove todos os dados da tabela de forma rápida, mas mantém a estrutura (colunas, índices, etc.).

a) Certo

b) Errado

✅ Gabarito: a) Certo

JOIN é essencial para consultar dados relacionais. Tipos comuns: INNER JOIN, LEFT JOIN, RIGHT JOIN.

10. A cláusula HAVING em SQL é usada para filtrar grupos após a agregação, enquanto WHERE filtra linhas antes.

a) Certo

b) Errado

✅ Gabarito: a) Certo
